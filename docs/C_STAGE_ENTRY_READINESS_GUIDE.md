# C단계 진입 준비 안내서

## 이번 단계의 목적

C단계에서는 Kling, Seedance 같은 외부 영상 서비스를 사용합니다. 외부 서비스는 계정, 비용, 사용 권리, API 키가 관련되므로 준비되지 않은 상태에서 자동 호출하면 안 됩니다.

이번에 만든 준비 점검 도구는 실제 영상을 생성하지 않습니다. 외부 연결 전에 안전 조건이 갖춰졌는지만 검사하고 한글 보고서를 만듭니다.

## 초보자에게 가장 중요한 개념

`진입 차단`은 실패가 아닙니다.

자동차가 출발하기 전에 안전벨트와 연료를 확인하는 것과 같습니다. 비용 승인이나 서비스 선택이 끝나지 않았다면 시스템이 스스로 멈추는 것이 정상입니다.

## 실행 방법

```powershell
& '.\.tools\python312\python.exe' -m tools.audit_c_stage_readiness
```

실행 결과는 `reports/C_STAGE_READINESS_REPORT.md`에서 확인합니다.

## 설정 파일

준비 상태는 `config/C_STAGE_READINESS.json`에 기록합니다.

중요한 규칙은 다음과 같습니다.

1. API 키 원문을 파일에 적지 않습니다.
2. API 키를 읽을 환경 변수 이름만 적습니다. 예: `ARK_API_KEY`
3. 첫 외부 연결은 반드시 1컷 dry-run으로 제한합니다.
4. 비용 상한과 외부 호출 승인이 있어야 합니다.
5. 첫 연결에서는 실제 인물 얼굴 자산을 사용하지 않습니다.
6. Kling API 방식은 공식 API 접근 권한을 확인한 경우에만 선택합니다.

## 공식 문서에서 반영한 기준

### Seedance

- API 키, 모델 활성화, 선불 리소스 팩 준비가 필요합니다.
- 첫 프레임, 첫 프레임과 마지막 프레임, 멀티모달 참조 방식은 서로 다른 생성 방식이므로 하나를 명확히 선택해야 합니다.
- 이미지, 영상, 오디오 참조에는 개수, 길이, 용량 제한이 있습니다.
- 실제 인물 얼굴이 포함된 자산은 권리와 업로드 제한을 먼저 확인해야 합니다.
- 생성 작업은 비동기 상태로 진행되므로 이후 연결 단계에서 대기, 성공, 실패, 만료 상태를 처리해야 합니다.

공식 문서:

- [Seedance 2.0 API 가이드](https://docs.byteplus.com/en/docs/modelark/1520757)
- [Seedance 2.0 프롬프트 가이드](https://docs.byteplus.com/en/docs/ModelArk/2222480)
- [Seedance 시작 튜토리얼](https://docs.byteplus.com/en/docs/ModelArk/2291680)
- [ModelArk 모델 활성화 안내](https://docs.byteplus.com/en/docs/ModelArk/1159200)

### Kling

- VIDEO 3.0은 멀티샷, 요소 참조, 네이티브 오디오 같은 기능을 제공합니다.
- 해상도와 네이티브 오디오 사용 여부에 따라 크레딧 비용이 달라집니다.
- 현재 확인한 공식 문서는 웹 사용 가이드입니다. 공식 API 접근이 별도로 확인되기 전에는 `web_ui` 방식으로 기록합니다.

공식 문서:

- [Kling AI VIDEO 3.0 공식 사용자 가이드](https://app.klingai.com/cn/quickstart/klingai-video-3-model-user-guide)

## 지금 보고서가 차단 상태인 이유

기본 설정은 일부러 다음 값을 승인하지 않은 상태로 둡니다.

- 외부 호출 승인
- 테스트 예산
- 사용할 영상 서비스
- 해당 서비스 계정과 모델 준비

다음 단계에서는 이 보고서를 함께 검수한 뒤, 사용할 첫 영상 서비스를 하나만 선택합니다. 선택 후에도 바로 여러 컷을 만들지 않고 1컷 dry-run만 진행합니다.
