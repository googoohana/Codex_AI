# B단계 첫 번째 도구: 에피소드 자동 검수

## 이번 단계에서 만든 것

`tools/validate_episode.py`는 에피소드 문서를 읽고 다음 제작 단계로 넘겨도 되는지 검사하는 로컬 도구입니다.

이 도구는 Kling, Seedance, GPT 같은 외부 AI를 호출하지 않습니다. 파일을 읽고 규칙을 확인한 뒤 한글 리포트를 만드는 안전한 반자동 도구입니다.

## 왜 먼저 검수 도구를 만드는가

자동화는 잘못된 작업도 빠르게 반복할 수 있습니다.

그래서 생성 도구보다 검수 도구를 먼저 만듭니다. 사람이 매번 눈으로 확인하던 반복 항목을 컴퓨터가 먼저 검사하고, 사람은 이야기와 영상 품질처럼 판단이 필요한 부분에 집중합니다.

## 현재 자동으로 검사하는 항목

1. `SHOT_PLAN` 파일이 있는가
2. 전체 컷 수는 몇 개인가
3. 모든 컷에 계획 길이가 있는가
4. 모든 컷이 이미지 생성 준비 상태인가
5. `IMAGE_PROMPTS` 수가 컷 수와 같은가
6. 각 컷에 영상 모델, 생성 방식, 참조 자산 정보가 있는가
7. 각 컷에 권리 정책, 생성 위험도, 실패 시 대안이 있는가

## 실행 방법

저장소 최상위 폴더에서 아래 명령을 실행합니다.

```powershell
& '.\.tools\python312\python.exe' -m tools.validate_episode 'episodes\EP01_sample'
```

현재 PC에서는 시스템 Python 대신 저장소 내부의 공식 Python 3.12.10 휴대용 버전을 사용합니다. `.tools` 폴더는 GitHub에 올리지 않습니다.

## 결과 확인

실행이 끝나면 아래 파일이 만들어집니다.

```text
episodes/EP01_sample/story_sync_audit/AUTO_REVIEW_REPORT.md
```

리포트 판정이 `통과`이면 문서 구조 자동 검사는 통과한 것입니다.

`수정 필요`이면 리포트의 `발견된 문제`를 고친 뒤 같은 명령을 다시 실행합니다.

## 꼭 기억할 점

자동 검수 통과는 영상 품질이 완벽하다는 뜻이 아닙니다.

자동 검수는 빠진 항목과 구조 오류를 찾습니다. 감정 표현, 이야기의 재미, 이미지 품질처럼 사람이 판단해야 하는 부분은 이후 수동 검수에서 확인합니다.

## 이번 단계의 학습 포인트

```text
입력 문서 -> 규칙 검사 -> 한글 리포트 -> 사람이 판단
```

이 흐름이 반자동 시스템의 기본 구조입니다. 컴퓨터가 반복 검사를 맡고, 사람은 최종 결정을 내립니다.
