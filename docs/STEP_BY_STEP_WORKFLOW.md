# 단계별 작업 흐름

## 전체 흐름

```text
A단계: 구조 만들기
B단계: 반자동 도구 만들기
C단계: 외부 AI 도구 연결하기
```

이 시스템은 A, B, C를 한 번에 만들지 않습니다. 먼저 사람이 이해할 수 있는 구조를 만들고, 그 다음 자동화 도구를 붙이고, 마지막에 외부 AI 도구를 연결합니다.

## A단계 흐름

1. 폴더 구조를 만든다.
2. 초보자 안내 문서를 만든다.
3. 반복 사용 템플릿을 만든다.
4. `EP01_sample` 샘플 에피소드를 만든다.
5. 체크리스트로 검수한다.
6. 사용자 승인을 받고 멈춘다.

A단계의 목표는 실제 영상을 만드는 것이 아닙니다. 앞으로 매번 같은 방식으로 영상을 만들 수 있도록 제작 질서를 만드는 것입니다.

## B단계 흐름

1. 에피소드 폴더 생성 스크립트를 만든다.
2. 컷 목록 생성 도구를 만든다.
3. `shot_sync_map.json` 생성 도구를 만든다.
4. 검수 리포트 초안을 만든다.
5. 실행 결과를 확인하고 멈춘다.

B단계의 목표는 반복 작업을 줄이는 것입니다. 하지만 외부 AI를 바로 호출하지는 않습니다. 먼저 로컬에서 안전하게 돌아가는 반자동 도구를 만듭니다.

## C단계 흐름

1. 이미지 생성 도구 연결을 준비한다.
2. 캐릭터와 장소 참조 자산을 정리한다.
3. 사용할 영상 모델과 생성 모드를 컷별로 확정한다.
4. 영상 생성 도구 연결을 준비한다.
5. TTS 생성 도구 연결을 준비한다.
6. 자막과 FFmpeg 렌더링을 연결한다.
7. 전체 검수 후 `DOWNLOAD_READY`에 저장한다.

C단계는 가장 나중에 진행합니다. API 키, 계정, 사용량, 비용, 실패 처리까지 고려해야 하기 때문입니다.

영상 모델 공식 문서는 바뀔 수 있으므로 C단계 시작 직전에 Kling, Seedance, Grok의 최신 공식 가이드와 제한을 다시 확인합니다.

## 매 단계의 멈춤 규칙

각 단계는 결과물을 만든 뒤 바로 다음 단계로 넘어가지 않습니다. 반드시 사용자가 열어보고 이해한 뒤 진행합니다.

검수할 때는 아래 질문을 사용합니다.

1. 이 단계의 결과물이 어디에 있는가
2. 파일 이름과 폴더 이름이 규칙에 맞는가
3. 초보자가 읽고 이해할 수 있는가
4. 다음 단계로 넘기면 문제가 커질 부분은 없는가
5. 개선할 설명이나 예시가 필요한가

## 좋은 진행 방식

처음에는 느리게 진행하는 것이 좋습니다.

영상 자동화 시스템은 처음 구조가 약하면 나중에 고치기 어렵습니다. 그래서 A단계에서는 속도보다 이해와 검수가 더 중요합니다.
