# 단계별 작업 흐름

## 전체 흐름

```text
A단계: 구조 만들기
B단계: 반자동 도구 만들기
C단계: 외부 AI 도구 연결하기
```

이 시스템은 A, B, C를 한 번에 만들지 않습니다. 먼저 사람이 이해할 수 있는 구조를 만들고, 그 다음 자동화 도구를 붙이고, 마지막에 외부 AI 도구를 연결합니다.

## A단계 흐름

1. 폴더 구조를 만든다.
2. 초보자 안내 문서를 만든다.
3. 반복 사용 템플릿을 만든다.
4. `EP01_sample` 샘플 에피소드를 만든다.
5. 체크리스트로 검수한다.
6. 사용자 승인을 받고 멈춘다.

A단계의 목표는 실제 영상을 만드는 것이 아닙니다. 앞으로 매번 같은 방식으로 영상을 만들 수 있도록 제작 질서를 만드는 것입니다.

## B단계 흐름

1. 에피소드 폴더 생성 스크립트를 만든다.
2. 컷 목록 생성 도구를 만든다.
3. `shot_sync_map.json` 생성 도구를 만든다.
4. 검수 리포트 초안을 만든다.
5. 실행 결과를 확인하고 멈춘다.

B단계의 목표는 반복 작업을 줄이는 것입니다. 하지만 외부 AI를 바로 호출하지는 않습니다. 먼저 로컬에서 안전하게 돌아가는 반자동 도구를 만듭니다.

### B단계 첫 번째 도구

현재 첫 번째 로컬 도구는 `tools/validate_episode.py`입니다.

이 도구는 `SHOT_PLAN`과 `IMAGE_PROMPTS`를 읽고 컷 수, 길이, 검수 상태, 모델 준비 필드가 서로 맞는지 검사합니다. 결과는 `story_sync_audit/AUTO_REVIEW_REPORT.md`에 저장합니다.

자동 검수가 통과해도 바로 외부 AI 연결로 넘어가지 않습니다. 사용자가 리포트와 도구 동작을 이해하고 검수한 뒤 다음 B단계 도구를 진행합니다.

### B단계 두 번째 도구

두 번째 로컬 도구는 `tools/create_episode.py`입니다.

이 도구는 템플릿에서 새 에피소드 문서와 표준 결과 폴더를 만들고, 문서 이름과 내부의 `EP##` 표시를 실제 에피소드 번호로 변경합니다. 기존 에피소드가 있으면 덮어쓰지 않고 멈춥니다.

실제 실행 예시로 `episodes/EP02`를 만들었습니다. EP02는 아직 빈 템플릿 상태이므로 자동 검수 결과가 `수정 필요`인 것이 정상입니다.

### B단계 세 번째 도구

세 번째 로컬 도구는 `tools/generate_cut_list.py`입니다.

이 도구는 `SCRIPT`의 대본 블록, 예상 길이, 내레이션, 대응 컷을 읽어 `story_sync_audit/CUT_LIST_DRAFT.md`를 만듭니다. 대응 컷이 없는 블록은 임시 컷 ID와 `수동 분할 필요` 상태로 표시합니다.

컷 목록 초안은 사람이 검토하기 위한 자료입니다. 기존 `SHOT_PLAN`을 수정하거나 자동으로 다음 단계에 넘기지 않습니다.

### B단계 네 번째 도구

네 번째 로컬 도구는 `tools/generate_shot_sync_map.py`입니다.

이 도구는 자동 검수를 통과한 `SHOT_PLAN`을 읽어 `story_sync_audit/shot_sync_map.json`을 만듭니다. JSON에는 컷별 설계 정보와 이미지, 영상, TTS, 자막의 예상 저장 경로가 들어갑니다.

EP01은 JSON 생성에 성공했지만, 아직 빈 템플릿 상태인 EP02는 자동 검수를 통과하지 못했으므로 JSON 생성이 차단됩니다.

### B단계 다섯 번째 도구

다섯 번째 로컬 도구는 `tools/run_local_pipeline.py`입니다.

이 도구는 컷 목록 초안 생성, 자동 검수, `shot_sync_map.json` 생성을 순서대로 실행합니다. 수동 분할 필요 블록이나 검수 문제가 있으면 즉시 멈추고 `PIPELINE_RUN_REPORT.md`에 이유를 기록합니다.

EP01은 전체 로컬 파이프라인을 완료했습니다. EP02는 수동 분할 필요 블록이 남아 있어 컷 목록 단계에서 안전하게 멈췄습니다.

## C단계 흐름

1. 외부 연결 전 준비 점검을 실행한다.
2. 이미지 생성 도구 연결을 준비한다.
3. 캐릭터와 장소 참조 자산을 정리한다.
4. 사용할 영상 모델과 생성 모드를 컷별로 확정한다.
5. 영상 생성 도구 연결을 준비한다.
6. TTS 생성 도구 연결을 준비한다.
7. 자막과 FFmpeg 렌더링을 연결한다.
8. 전체 검수 후 `DOWNLOAD_READY`에 저장한다.

C단계는 가장 나중에 진행합니다. API 키, 계정, 사용량, 비용, 실패 처리까지 고려해야 하기 때문입니다.

영상 모델 공식 문서는 바뀔 수 있으므로 C단계 시작 직전에 Kling, Seedance, Grok의 최신 공식 가이드와 제한을 다시 확인합니다.

### C단계 첫 번째 도구

첫 번째 C단계 도구는 `tools/audit_c_stage_readiness.py`입니다.

이 도구는 `config/C_STAGE_READINESS.json`을 읽고 외부 호출 승인, 1컷 dry-run 제한, 테스트 예산, 권리 정책, Kling과 Seedance 준비 상태를 검사합니다. API 키 원문이 설정 파일에 들어가면 차단합니다.

결과는 `reports/C_STAGE_READINESS_REPORT.md`에 저장합니다. 현재 기본 설정은 외부 호출과 예산을 승인하지 않았기 때문에 `진입 차단`이 정상 결과입니다.

## 매 단계의 멈춤 규칙

각 단계는 결과물을 만든 뒤 바로 다음 단계로 넘어가지 않습니다. 반드시 사용자가 열어보고 이해한 뒤 진행합니다.

검수할 때는 아래 질문을 사용합니다.

1. 이 단계의 결과물이 어디에 있는가
2. 파일 이름과 폴더 이름이 규칙에 맞는가
3. 초보자가 읽고 이해할 수 있는가
4. 다음 단계로 넘기면 문제가 커질 부분은 없는가
5. 개선할 설명이나 예시가 필요한가

## 좋은 진행 방식

처음에는 느리게 진행하는 것이 좋습니다.

영상 자동화 시스템은 처음 구조가 약하면 나중에 고치기 어렵습니다. 그래서 A단계에서는 속도보다 이해와 검수가 더 중요합니다.
