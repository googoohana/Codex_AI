# B단계 다섯 번째 도구: 로컬 파이프라인 실행기

## 이번 단계에서 만든 것

`tools/run_local_pipeline.py`는 지금까지 만든 B단계 도구를 안전한 순서로 실행합니다.

```text
컷 목록 초안 -> 자동 검수 -> shot_sync_map.json
```

중간 단계에 문제가 있으면 다음 단계로 넘어가지 않고 멈춥니다.

## 실행 방법

처음 실행하거나 기존 생성 파일을 보호하려면 아래 명령을 사용합니다.

```powershell
& '.\.tools\python312\python.exe' -m tools.run_local_pipeline 'episodes\EP01_sample'
```

기존 생성 산출물을 검토한 뒤 최신 문서 기준으로 갱신하려면 `--refresh-generated`를 명시합니다.

```powershell
& '.\.tools\python312\python.exe' -m tools.run_local_pipeline 'episodes\EP01_sample' --refresh-generated
```

## 왜 갱신 옵션이 필요한가

컷 목록과 JSON을 자동으로 덮어쓰면 사람이 검토하던 결과가 사라질 수 있습니다.

그래서 기본 실행은 기존 생성 파일이 있으면 멈춥니다. 사용자가 갱신 의도를 분명히 표시했을 때만 생성 산출물을 다시 만듭니다.

## 파이프라인 멈춤 규칙

1. 기존 컷 목록이 있고 갱신 옵션이 없으면 컷 목록 단계에서 멈춥니다.
2. 수동 분할 필요 블록이 있으면 컷 목록 단계에서 멈춥니다.
3. 자동 검수를 통과하지 못하면 검수 단계에서 멈춥니다.
4. 검수 통과 후에만 `shot_sync_map.json`을 만듭니다.
5. 기존 동기화 지도가 갱신되지 않았다면 리포트에 주의 문구를 남깁니다.

## 실행 결과 확인

모든 실행 결과는 아래 파일에 기록됩니다.

```text
story_sync_audit/PIPELINE_RUN_REPORT.md
```

리포트에는 최종 상태, 멈춘 단계, 단계별 결과, 다음에 확인할 문제가 기록됩니다.

## 실제 실행 결과

- EP01: 컷 목록 6개, 자동 검수 통과, JSON 생성 완료
- EP02: 수동 분할 필요 블록 3개 발견, 컷 목록 단계에서 중간 차단

EP02의 중간 차단은 실패가 아니라 안전장치가 작동한 결과입니다.

## 현재 B단계 상태

현재 B단계의 기본 로컬 흐름이 연결되었습니다.

```text
새 에피소드 생성
-> 컷 목록 초안 생성
-> 자동 검수
-> shot_sync_map.json 생성
-> 실행 리포트 확인
```

아직 GPT, Kling, Seedance, Grok, TTS, FFmpeg 같은 외부 도구는 호출하지 않습니다.

## 이번 단계의 학습 포인트

자동화 파이프라인은 모든 작업을 무조건 끝까지 실행하는 프로그램이 아닙니다.

각 단계의 결과를 확인하고, 문제가 있으면 정확한 위치에서 멈추는 것이 좋은 시스템의 핵심입니다.
